GitHub 저장소에 해당 파일이 없어서 실패했습니다.

수정 방법

1. update_kospi.py를 GitHub 저장소의 가장 첫 화면(최상위)에 업로드합니다.
   폴더 안에 넣지 마세요.

2. requirements.txt도 저장소 최상위에 업로드합니다.

3. main.yml의 내용을 새 main.yml 내용으로 전부 교체합니다.
   GitHub에서 위치는 .github/workflows/main.yml입니다.

4. 기존에 업로드한 docs/index.html은 그대로 둡니다.

5. GitHub Actions에서 Run workflow를 다시 실행합니다.

  update_kospi.py
